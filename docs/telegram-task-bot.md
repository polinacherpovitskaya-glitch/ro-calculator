# Telegram Task Bot

Отдельный бот для уведомлений по задачам.

## Что умеет

- присылает уведомление, когда сотруднику назначили задачу;
- присылает уведомление, когда меняется статус его задачи;
- присылает уведомление наблюдателям, если статус watched-задачи изменился;
- присылает уведомление, когда задачу отправили на согласование;
- присылает уведомление при упоминании в комментарии;
- использует ту же привязку сотрудника по `telegram_id`, что и основной `timebot`.

## Команды

- `/start` — привязать Telegram к сотруднику;
- `/status` — проверить, к кому привязан бот;
- `/help` — краткая справка.

## Переменные окружения

Смотри [bot/.env.example](/Users/krollipolli/Documents/Github/RO%20calculator/bot/.env.example)

- `TASK_BOT_TOKEN` — токен отдельного task-bot;
- `SUPABASE_URL`
- `SUPABASE_KEY`
- `TASK_BOT_POLL_INTERVAL_MS` — как часто опрашивать очередь уведомлений.

Для `timebot`:

- `ENABLE_TASK_NOTIFICATION_WORKER=false`

Это важно, чтобы два бота не обрабатывали одну и ту же очередь одновременно.

## Запуск

```bash
cd bot
npm install
npm run start:task
```

## Источник уведомлений

UI пишет события в `task_notification_events`, а task-bot забирает только необработанные записи (`processed_at is null`) и помечает их обработанными после отправки.
