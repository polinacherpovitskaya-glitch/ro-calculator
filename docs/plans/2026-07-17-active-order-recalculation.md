# План: пересчёт активных заказов по текущим нормам

Связанная спека: [`../specs/2026-07-17-active-order-recalculation.md`](../specs/2026-07-17-active-order-recalculation.md)

## 1. Единая норма запуска и защита кастомов

- [ ] В движке выбрать норму бланка/кастома, если нет индивидуального override.
- [ ] Учесть 2 ч на каждую кастомную форму и 0,5 ч на каждый цвет; для бланка
  оставить только 0,5 ч на цвет.
- [ ] Не применять каталоговую скорость к кастому только из-за `template_id`.
- [ ] Закрыть правила unit/smoke-тестом.

## 2. Воспроизводимый пересчёт первой волны

- [ ] Добавить скрипт dry-run для образцов и production со строгой
  валидацией снимков и агрегированным отчётом.
- [ ] Добавить `--apply`: локальный backup, обновление шапки и расчётных
  полей позиций, повторное чтение и проверка результата.
- [ ] Сделать отдельный scope для черновиков, не применяя его в первой волне.
- [ ] Добавить тест скрипта на бланк, кастом и скидку.

## 3. Понятная шкала квартальной загрузки

- [ ] Добавить чистую функцию трёх месяцев квартала и тест её границ.
- [ ] Показать две тонкие чёрные границы, подписи месяцев и текущий месяц.
- [ ] Убрать из основного текста неоднозначное «сегодня N%».

## 4. Проверка и выпуск

- [ ] Выполнить dry-run на production и проверить, что нет unsafe-позиций.
- [ ] Поднять версию и cache-bust изменённых браузерных файлов.
- [ ] Прогнать syntax, unit/smoke и production preflight.
- [ ] Выпустить код; затем применить первую волну и повторно проверить данные.
