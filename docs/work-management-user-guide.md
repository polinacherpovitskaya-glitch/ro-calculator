# Инструкция: задачи и проекты

Короткая инструкция для команды по работе с новым внутренним модулем задач и проектов.

## 1. Где это находится

- `Задачи` в левом меню: общий рабочий центр задач.
- `Проекты` в левом меню: список инициатив, съёмок, акций, дизайна, сайта и других внутренних проектов.
- `Заказы` → карточка заказа → вкладка задач: задачи и проекты, связанные с конкретным заказом.

Если после обновления сайта виден старый экран, сделайте жёсткое обновление страницы: `Ctrl+Shift+R`.

## 2. С чего начинать

Открывайте `Задачи`, если нужно:

- поставить разовую задачу;
- посмотреть личную очередь;
- отследить дедлайны;
- согласовать или закрыть работу.

Открывайте `Проекты`, если работа состоит из нескольких связанных задач:

- съёмка;
- маркетинговая акция;
- контент / SMM;
- дизайн;
- закупка;
- сайт / разработка;
- внутренний запуск;
- другая инициатива.

## 3. Как создать задачу

В разделе `Задачи`:

1. Нажмите `+ Задача`.
2. Либо нажмите `Из шаблона`, если задача типовая.
3. Заполните обязательные поля:
   - `Название`
   - `Исполнитель`
   - `Дедлайн`
   - хотя бы один контекст: `Заказ`, `Проект` или `Направление`
4. Нажмите `Сохранить`.

Минимум, который нужно заполнять всегда:

- что нужно сделать;
- кто делает;
- до какого срока;
- к чему относится задача.

## 4. Какие бывают контексты задачи

- `Заказ`: задача относится к конкретному коммерческому заказу.
- `Проект`: задача относится к внутреннему или заказному проекту.
- `Направление`: задача относится к зоне ответственности без заказа и без проекта.

Доступные направления:

- Маркетинг
- Дизайн
- Склад
- China
- Сайт
- Финансы
- Общее

## 5. Как работать в разделе "Задачи"

Вверху страницы есть 3 среза:

- `Мои задачи`
- `Все задачи`
- `Просроченные`

И 3 представления:

- `Список`
- `Канбан`
- `Календарь`

Рекомендуемый режим по умолчанию:

- `Мои задачи` + `Список` для личной очереди;
- `Все задачи` + `Канбан` для обзора команды;
- `Просроченные` для разбора просрочек;
- `Календарь` для дедлайнов по датам.

## 6. Как пользоваться карточкой задачи

Слева показывается список, канбан или календарь. Справа открывается карточка выбранной задачи.

В карточке можно:

- менять `статус`;
- менять `приоритет`;
- менять `исполнителя`, `постановщика`, `проверяющего`;
- менять `дедлайн` и время;
- выбирать `заказ`, `проект`, `направление`;
- вести `описание`;
- заполнять поле `Ждём от кого / что ждём`;
- добавлять `наблюдателей`;
- добавлять `чек-лист`;
- создавать `подзадачи`;
- писать `комментарии`;
- прикладывать `ссылки` и `файлы`;
- смотреть `историю`.

## 7. Статусы задачи

- `Входящие`: задача только поступила.
- `Запланировано`: задача согласована, но ещё не стартовала.
- `В работе`: исполнитель работает.
- `На согласовании`: исполнитель передал результат проверяющему.
- `Ждём`: есть внешний блокер.
- `Готово`: задача завершена.
- `Отменено`: задача больше не актуальна.

## 8. Как работает согласование

Если по задаче нужен review:

1. Укажите `Проверяющий`.
2. Исполнитель переводит задачу кнопкой `На согласование`.
3. Проверяющий открывает карточку задачи.
4. Дальше у него 2 действия:
   - `Вернуть в работу`
   - `Готово`

Все эти действия попадают в историю задачи.

## 9. Как не терять обсуждение

Внутри задачи используйте:

- `Комментарии` для обсуждения;
- `Файлы и ссылки` для референсов, макетов, документов и исходников;
- `Чек-лист` для списка приёмки;
- `Подзадачи`, если работу нужно разбить.

Упоминания в комментариях пишите как `@Имя Фамилия`.

## 10. Ручная очередь исполнителя

Для дизайнера и других ролей с плотной очередью есть ручной порядок задач.

Как пользоваться:

1. Откройте `Задачи`.
2. Переключитесь в `Список`.
3. В поле сортировки выберите `Ручная очередь`.
4. Используйте кнопки `↑` и `↓` у задач.

Так можно быстро собрать реальный приоритет дня, не теряя задачи в сообщениях.

## 11. Как пользоваться фильтрами

Фильтры в задачах позволяют быстро собрать нужный срез:

- по статусу;
- по приоритету;
- по исполнителю;
- по постановщику;
- по проекту;
- по заказу;
- по направлению;
- по дедлайну;
- `Только мои`;
- `Только на согласовании`;
- `Только "Ждём"`.

Поиск ищет по:

- названию задачи;
- описанию;
- проекту;
- заказу;
- комментариям.

## 12. Как создавать проекты

В разделе `Проекты`:

1. Нажмите `+ Проект`.
2. Или `Из шаблона`.
3. Заполните:
   - название;
   - тип;
   - владелец;
   - статус;
   - при необходимости связанный заказ;
   - направление;
   - даты;
   - краткое описание;
   - цель;
   - результат.
4. Нажмите `Сохранить проект`.

Проект может быть:

- без заказа;
- привязан к одному заказу.

У одного заказа может быть несколько проектов.

## 13. Что есть внутри проекта

В карточке проекта доступны:

- основные поля проекта;
- связанные задачи;
- кнопка `+ Задача` для создания задачи сразу в контексте проекта;
- файлы и ссылки;
- активность по проекту.

Если проект привязан к заказу, его задачи видны и в проекте, и в карточке заказа.

## 14. Как работать из карточки заказа

В карточке заказа на вкладке задач видно:

- прямые задачи заказа;
- задачи проектов, которые привязаны к этому заказу;
- связанные проекты.

Из карточки заказа можно сразу создать:

- `+ Задача`
- `+ Проект`

В этом случае нужный контекст подставляется автоматически.

## 15. Рекомендуемый рабочий процесс

Для менеджеров:

- не писать новую задачу только в чат;
- сразу создавать задачу в системе;
- прикладывать ссылку, файл, описание и дедлайн;
- если работа большая, сначала создавать проект, а внутри него уже задачи.

Для исполнителей:

- работать из `Моих задач`;
- держать актуальный статус;
- всё обсуждение оставлять в комментариях задачи;
- сдавать задачу через `На согласовании`, если нужен review.

Для проверяющих:

- открывать задачи со статусом `На согласовании`;
- возвращать в работу с комментарием, если нужно;
- переводить в `Готово`, если результат принят.

## 16. Что важно помнить

- Сейчас все сотрудники видят все задачи.
- Приватных задач в этой версии нет.
- Главный принцип: задача должна жить в системе, а не в чате.
- Если по работе много шагов и участников, лучше заводить проект.
