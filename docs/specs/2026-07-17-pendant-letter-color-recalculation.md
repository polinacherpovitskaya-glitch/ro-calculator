# Цветные партии букв и пересчёт активных заказов

## Цель

Буквы в одном подвесе производятся отдельными партиями по цветам. Каждый цвет
бланка требует свой запуск на 0,5 часа. После исправления формулы необходимо
сохранить новый расчёт только для образцов и заказов в производстве; черновики
остаются следующей волной.

## Формула букв

- Количество букв одного цвета равно числу таких символов в подвесе, умноженному
  на тираж подвесов.
- Каждая цветовая группа рассчитывается отдельной бланковой партией: её
  фактическая производительность, срезание литника и один запуск цвета.
- Группы суммируются по себестоимости, часам и финансовому breakdown. Для
  старого подвеса без цвета остаётся одна группа, поэтому его поведение не
  меняется.
- Пример `ELEMENT`: 50 подвесов, шесть белых и одна красная буква — 300 белых
  и 50 красных. Это два запуска по 0,5 ч, а не один запуск, масштабированный
  вместе с тиражом.

## Первая волна данных

Статусы: `sample`, `production`, `production_casting`, `production_printing`,
`production_hardware`, `production_packaging`, `in_production`, `delivery`.

Перед записью выполняется dry-run. Скрипт создаёт локальный backup, меняет
только расчётные поля шапки и позиций, сохраняет статусы/цены/состав/склад и
после каждой записи перечитывает результат. Составляется отчёт «было → стало»
по каждому заказу; он проверяется перед применением.
