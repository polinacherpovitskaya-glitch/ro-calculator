# Редизайн калькулятора — UX улучшения

**Дата:** 2026-04-02
**Статус:** Одобрено к реализации

## Контекст

Калькулятор заказов используется постоянно для двух основных сценариев:
1. Создание нового заказа с нуля
2. Редактирование существующего заказа

Текущие проблемы: нет визуальной иерархии, карточки изделий громоздкие, непонятно в каком порядке заполнять, нет кнопок "свернуть/развернуть", кнопки не всегда в логичных местах.

**Ограничение:** Не ломать логику склада (warehouse), не менять JS-логику калькулятора — только HTML-структуру и CSS.

---

## Подход: Прогрессивное раскрытие (B1)

Та же одна страница, но с нумерованными секциями, скрытыми редкими полями и чёткими кнопками в правильных местах. Минимальный риск сломать существующую логику.

---

## Дизайн

### 1. Структура страницы — нумерованные секции

Заголовок страницы + три пронумерованных секции:

```
[Заголовок: "Новый заказ" / название заказа]  [Сбросить] [💾 Сохранить]

① О заказе
② Изделия          ← главная рабочая зона, всегда выделена
③ Фурнитура и упаковка

[итог + скидка + кнопки действий]
```

Каждая секция — белая карточка с кружком-номером в заголовке. Секция ② имеет синюю рамку чтобы привлечь внимание (это главное действие).

### 2. Секция ① — О заказе

**Всегда видны** (2 колонки):
- Название заказа
- Клиент
- Менеджер (dropdown)
- Дедлайн (дата)
- Заметки (textarea, полная ширина)

**Скрыты под ссылками** (раскрываются кликом):
- "▸ Контакты и ссылки" → адрес доставки, Telegram, CRM, Fintablo
- "▸ Юр. данные (для КП)" → юр. название, ИНН, адрес, банк, р/с, БИК

Это убирает 6–8 редко нужных полей из основного вида.

### 3. Секция ② — Изделия

**Пустое состояние:**
- Иконка 📦, текст "Добавьте изделие чтобы начать расчёт"
- Крупная синяя кнопка "＋ Добавить изделие"
- Мелкая ссылка "или добавить подвес из букв"

**Каждая карточка изделия — два состояния:**

*Свёрнуто (по умолчанию):*
- Шапка: бейдж "Изделие N", название, кнопки "▼" и "✕ Удалить"
- Строка резюме: количество | цвет (кружок + название) | кол-во печати | цена/шт
- Ссылка "Развернуть ▼" справа

*Развёрнуто (после клика):*
- Шапка: бейдж, название, кнопки "▲ Свернуть" и "✕ Удалить"
- **Раздел ОСНОВНОЕ:**
  - Выбор изделия (форма) с превью фото
  - Количество / Вес (г) / Шт/час — три колонки
  - Цвета: теги с кружком цвета + кнопка "+ цвет"
  - Печать: таблица (кол-во / цена/шт / доставка) + ссылка "+ ещё печать"
- **Раздел ДОПОЛНИТЕЛЬНО** (свёрнут по умолчанию, раскрывается кликом):
  - Чекбоксы: Сложный дизайн / NFC-чип / Своя пресс-форма / Доставка включена
  - Доп. поля NFC (появляются при включении чекбокса)
- **Зелёная строка внизу карточки:**
  - Себестоимость: XX ₽/шт | Цена продажи: [поле] | → XX% маржа

**Кнопка "+ Добавить изделие"** — широкая, с пунктирной рамкой, после последней карточки.

### 4. Секция ③ — Фурнитура и упаковка

Две подсекции: ФУРНИТУРА и УПАКОВКА.

Каждая позиция — строка:
- Фото/иконка (28×28px)
- Название + SKU + кол-во + цена/шт (мелко)
- Цена продажи (справа, зелёная)
- Кнопки ✏️ (редактировать) и ✕ (удалить)

Кнопки добавления: "＋ Фурнитура из склада" и "＋ Упаковка из склада" — серые, с пунктирной рамкой.

Секция может быть свёрнута целиком (клик по заголовку) если ничего не добавлено.

### 5. Итог + скидка + кнопки

Блок внизу страницы (не floating, обычный в потоке):

**Итог заказа:**
- 3 колонки: Выручка | Прибыль | Маржа%

**Скидка — встроена в итоговую строку:**
- Переключатель: [нет] [₽] [%]
- Поле ввода суммы/процента
- Отображение вычета и итоговой суммы

**Кнопки:**
- Слева: "Авто-сохранено в ЧЧ:ММ" (статус)
- Справа: [📄 Скачать КП] [💾 Сохранить заказ]

Плавающая кнопка сохранения (FAB) остаётся как есть — дополнительное удобство при скролле.

## Что НЕ меняется

- Вся JS-логика калькулятора (calculator.js, supabase.js)
- Логика склада (warehouse.js, warehouse_photos.js) — не трогаем
- Алгоритм расчёта себестоимости
- Структура данных и поля в БД
- Логика автосохранения
- Подвесы из букв (pendant.js) — UI не меняем, только встраиваем в новую структуру
- Все события (oninput, onchange, onclick) на существующих элементах

## Подход к реализации

**Только CSS + минимальные изменения HTML.**
Не переписывать JS. Добавить классы к существующим элементам, обернуть в новые div-контейнеры, добавить toggle-логику для сворачивания через минимальный JS.

Изменения в `index.html`:
- Обернуть секции калькулятора в нумерованные контейнеры
- Добавить кнопки collapse/expand к карточкам изделий
- Переместить скидку в блок итогов
- Добавить empty state для пустых секций
- Скрыть редкие поля (юр. данные, контакты) под раскрывающиеся ссылки

Изменения в `css/style.css`:
- Стили для нумерованных секций (`.calc-section`, `.calc-section-number`)
- Стили для свёрнутых/развёрнутых карточек (`.item-card-collapsed`, `.item-card-expanded`)
- Стили для строки резюме карточки (`.item-card-summary`)
- Стили для зелёной строки цены (`.item-card-pricing`)
- Стили для empty state (`.calc-empty-state`)
- Стили для строки фурнитуры (`.hw-row`)
- Обновить итоговый блок (`.calc-footer`)

Добавить в `app.js` (не создавать новый файл, чтобы не добавлять зависимостей):
- Логика toggle для карточек изделий (свернуть/развернуть)
- Логика toggle для секций "Контакты" и "Юр. данные"
- Логика toggle для секции "Дополнительно" внутри карточки

## Тесты

После реализации проверить вручную:
- Все существующие Playwright тесты в `/tests/` проходят без изменений
- Кнопка "Добавить изделие" создаёт карточку
- Карточка изделия сворачивается и разворачивается
- Поля формы сохраняют значения при сворачивании
- Фурнитура из склада добавляется (picker открывается)
- Расчёт себестоимости работает после изменений
- Автосохранение срабатывает
- Скидка применяется корректно
- Скачивание КП работает
