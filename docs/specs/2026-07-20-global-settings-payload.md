# Быстрая загрузка глобальных настроек

**Дата:** 2026-07-20  
**Статус:** approved for implementation

## Проблема

При старте `calc.recycleobject.ru` функция `loadSettings()` читает все строки
таблицы `settings`. В таблице есть рабочие снимки, которые не являются
глобальными настройками калькулятора: в частности, снимок FinTablo занимает
около 11,7 МБ, а снимок Точки — около 1,4 МБ. Из-за этого чтение превышает
пятисекундный таймаут в браузере.

Приложение продолжает работу на локальном fallback, но расчёт, загрузка и
настройки могут оказаться не теми, что сохранены в общей базе. В живом
интерфейсе это проявляется как `loadSettings exception: timeout`.

## Решение

`loadSettings()` читает только глобальные скалярные и плановые настройки. Из
общей выборки исключаются JSON-снимки модулей, у которых уже есть собственные
точечные загрузчики: финансы, банк, склад, готовая продукция, авторизация,
план-факт и work-management.

Сезонное распределение мощности остаётся в основной выборке: его используют
цены, калькулятор, настройки и производственная доска.

Yandex-зеркало перестаёт сначала загружать единый 18 МБ `bootstrap.json`.
Сборка публикует рядом маленькие файлы по ключам bootstrap, а браузер запрашивает
только нужные текущему экрану: на входе — `authAccounts`, для настроек —
настройки, для заказов — заказы и их позиции. Старый цельный файл сохраняется
лишь как совместимый fallback на время одного цикла выкладки.

## Границы

- Не меняются данные в Supabase и схемы таблиц.
- Не меняются отдельные загрузчики снимков модулей.
- Не меняется логика fallback при реальной недоступности общей базы.
- Запрос к `settings` остаётся одним запросом, но не передаёт большие и
  несвязанные с расчётом JSON-поля.
- Полный `bootstrap.json` не удаляется и не меняет состав данных; новые shard
  файлы только убирают его из критического пути холодного старта calc2.

## Критерии приёмки

- В общем запросе настроек отсутствуют все ключи выделенных JSON-снимков.
- `seasonal_load_percent_json` и `seasonal_load_plan_json` по-прежнему
  попадают в `App.settings`.
- Ответ общего запроса не содержит FinTablo/Точка/склад/авторизацию/work JSON.
- Полный набор локальных smoke-тестов и version smoke проходят.
- Новая static-сборка содержит bootstrap shard для каждого ключа, а calc2 при
  наличии shard не делает запрос к цельному `data/bootstrap.json`.
