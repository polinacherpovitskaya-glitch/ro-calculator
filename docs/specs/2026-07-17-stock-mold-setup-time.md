# Норма запуска для повторного кастомного молда

**Дата:** 2026-07-17  
**Статус:** approved for implementation

## Решение

Если базовый кастомный молд уже лежит на складе (`base_mold_in_stock: true`),
время его запуска — 0 ч. Его не надо заново готовить как новый молд.

Подготовка цвета не исчезает ни при каком повторе: 0,5 ч начисляется за каждый
уникальный цвет, потому что инжектор нужно очистить и переключить на цвет нового
заказа. Это же правило применяется к бланкам.

Итог для кастома без индивидуального legacy-override:

```
часы запуска = 2 × (новая базовая форма ? 1 : 0 + доп. новые молды) + 0,5 × число цветов
```

Индивидуальный `setup_hours_override` остаётся явным старым исключением и не
переписывается массово. Скорость `pieces_per_hour` кастомной позиции не меняется.

## Примеры

- Бланк в двух цветах: `2 × 0,5 = 1 ч`.
- Новый кастом в двух цветах: `2 + 2 × 0,5 = 3 ч`.
- Тот же кастом, базовый молд на складе, два цвета: `2 × 0,5 = 1 ч`.
- Складской базовый молд и два новых дополнительных, два цвета:
  `2 × 2 + 2 × 0,5 = 5 ч`.
