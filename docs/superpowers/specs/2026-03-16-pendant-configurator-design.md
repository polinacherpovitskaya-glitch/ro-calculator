# Конфигуратор подвесов из букв

## Проблема

Подвесы из букв — очень популярный товар, но сейчас для оформления одного подвеса менеджеру приходится:
- Каждую букву добавлять как отдельный product item (бланк "Буква А", "Буква К" и т.д.)
- Шнур и карабин добавлять как отдельные hardware items
- Печать — как отдельный printing на конкретной букве
- Лимит в 6 product items быстро исчерпывается на одну надпись

Это занимает много времени и провоцирует ошибки.

## Решение

Визард-модалка внутри калькулятора заказов. Кнопка «+ Подвес из букв» рядом с существующими кнопками добавления. Менеджер за 5 шагов собирает подвес, результат сохраняется как единая сущность (`item_type: 'pendant'`) в `calculator_data`.

## Компоненты подвеса

1. **Шнур** — со склада (готовый с силиконовым наконечником, или отдельно шнур + металлические наконечники) или кастом из Китая
2. **Карабин** — со склада или кастомный цвет
3. **Элементы** — буквы (кириллица/латиница) + спецэлементы (❤️, 😊). Все продаются по одной цене. Цена определяется по суммарному тиражу всех элементов из раздела бланков
4. **Печать** — на отдельных элементах (чаще на ❤️), стоимость вводится вручную (от типографии)
5. **Упаковка** — опционально, на весь подвес

## Ценообразование элементов

- Все буквы и спецэлементы стоят одинаково
- Цена за штуку берётся из раздела бланков по суммарному тиражу
- Суммарный тираж = количество элементов в надписи × количество подвесов (например: 6 элементов × 500 подвесов = 3000 шт → цена из бланков за тираж 3000)
- Менеджер может скорректировать цену вручную (скидка или наценка)

## Структура данных

### Хранение

Подвесы хранятся в `calculator_data` в отдельном массиве `pendants: [...]`, аналогично `hardwareItems` и `packagingItems`. Не занимают лимит product items (0-6). Количество подвесов в одном заказе не ограничено.

При сохранении в БД каждый подвес записывается как строка в `order_items` с `item_type: 'pendant'`. Все данные подвеса сериализуются в поле `item_data` (JSON blob), так же как это делают hardware и packaging items. Новых DB-колонок не требуется.

При восстановлении заказа (`_restoreOrderFromDB`) — items с `item_type: 'pendant'` загружаются в `this.pendants[]`.

### Модель данных подвеса

```js
{
  item_type: 'pendant',
  pendant_id: 'pnd_<timestamp>',
  name: 'КЭШШШ❤️',              // надпись целиком
  quantity: 500,                   // кол-во подвесов

  // Элементы (буквы + спецэлементы)
  elements: [
    { char: 'К', color: 'белый', has_print: false },
    { char: 'Э', color: 'белый', has_print: false },
    { char: 'Ш', color: 'голубой', has_print: false },
    { char: 'Ш', color: 'голубой', has_print: false },
    { char: 'Ш', color: 'голубой', has_print: false },
    { char: '❤️', color: 'красный', has_print: true, print_price: 15.50 }
  ],

  // Шнур
  cord: {
    source: 'warehouse',          // 'warehouse' | 'custom'
    warehouse_item_id: 'SLS-800-GR-NN-GR',
    name: 'Шнур с силик. наконечником зеленый',
    price_per_unit: 23,            // ₽ за шт (включая доставку для кастома)
    delivery_price: 0,             // доп. доставка ₽/шт (для кастома из Китая)
    assembly_speed: 20             // шт/час (скорость нанизывания букв на шнур)
  },

  // Карабин
  carabiner: {
    source: 'warehouse',
    warehouse_item_id: 'CR-RNG-023-SV',
    name: 'Круглый карабин 2.3 см серебряный',
    price_per_unit: 5,
    delivery_price: 0
  },

  // Упаковка (опционально)
  packaging: null,                // или { source, name, price_per_unit, delivery_price, assembly_speed }

  // Цены
  element_price_per_unit: null,   // авто из бланков, менеджер может переписать
  sell_price_override: null        // ручная цена за весь подвес
}
```

### Максимальная длина надписи

Рекомендуемый лимит: 20 элементов. При вводе более 20 — предупреждение (но не блокировка).

## Визард-модалка: 5 шагов

### Шаг 1: Надпись и количество
- Поле «Количество подвесов» (число)
- Поле «Надпись» (текстовый ввод)
- Кнопки быстрой вставки спецэлементов: `[❤️]` `[😊]`
- Под полем — визуальные «бусины» (кружки/плашки с буквами в ряд)

### Шаг 2: Цвета
- Бусины из шага 1, кликабельные
- Менеджер выделяет несколько бусин (клик / shift+клик) и назначает цвет группе
- По умолчанию все бусины серые (без цвета)
- Палитра — текстовое поле для ввода цвета (свободный текст: "белый", "голубой", "красный"). Не привязана к `app_colors` — у букв свои цвета пластика, менеджер вводит как есть

### Шаг 3: Печать
- Те же бусины, под каждой чекбокс «Печать»
- При отметке — поле «Стоимость печати ₽/шт»
- Обычно 1-2 элемента с печатью

### Шаг 4: Шнур и карабин
- Блок «Шнур»: переключатель «Со склада» / «Кастом»
  - Со склада → поиск по warehouse (категория cords), цена подтягивается
  - Кастом → название + цена вручную (или из каталога Китай)
- Блок «Карабин»: аналогично
  - Со склада → поиск по warehouse (категория carabiners)
  - Кастом → название + цена вручную

### Шаг 5: Упаковка + итого
- Опциональный блок упаковки (со склада / кастом)
- Сводка с расчётом:
  - Элементы: N шт × qty = суммарный тираж → цена из бланков (редактируемое поле)
  - Шнур: qty × цена
  - Карабин: qty × цена
  - Печать: qty × стоимость (для каждого элемента с печатью)
  - Упаковка: qty × цена (если есть)
  - **Итого за подвес / Итого за всё**
- Кнопка «Добавить в заказ»

Навигация: можно кликнуть на любой шаг и вернуться. Шаг 1 (надпись) обязателен.

## Отображение в калькуляторе

После добавления подвес появляется в калькуляторе как свёрнутая карточка:

```
┌─────────────────────────────────────────────┐
│ 🔤 Подвес "КЭШШШ❤️" × 500 шт    [✏️] [🗑️] │
│ Шнур зелёный силикон + карабин 2.3 см       │
│ 6 элементов, 1 с печатью                    │
│ Итого: ₽XXX за шт / ₽YYY                   │

```

Кнопка ✏️ — открывает визард заново для редактирования. Кнопка 🗑️ — удаляет подвес.

## Интеграция с расчётами

### Новая функция: `calculatePendantCost(pendant, params)`

Подвес рассчитывается отдельной функцией, возвращающей структуру, совместимую с `calculateOrderSummary`.

**Себестоимость одного подвеса (costPerUnit):**

```
elementsCost = elements.length * element_price_per_unit
cordCost     = cord.price_per_unit + cord.delivery_price
carabinerCost = carabiner.price_per_unit + carabiner.delivery_price
printCost    = SUM(elements where has_print: element.print_price)
packagingCost = packaging ? packaging.price_per_unit + packaging.delivery_price : 0

costPerUnit = elementsCost + cordCost + carabinerCost + printCost + packagingCost
```

**Цена продажи одного подвеса (sellPerUnit):**

Если `sell_price_override` задан — используется он. Иначе рассчитывается стандартно:
```
sellPerUnit = costPerUnit / (1 - taxRate - commercialRate - marginTarget)
```
где `marginTarget` определяется по `pendant.quantity` через `CALC_TIER_MARGINS`.

**Маржа:** `calculateActualMargin(sellPerUnit, costPerUnit)`

### Ценообразование элементов (element_price_per_unit)

Цена за один элемент определяется автоматически из справочника бланков:
1. Вычисляем суммарный тираж: `totalElements = elements.length * quantity`
2. Находим бланк-шаблон для букв (reference blank — хранится в настройках, `settings.pendant_letter_blank_id`)
3. Вычисляем себестоимость бланка для тиража `totalElements` через `calculateItemCost`
4. Получаем sell price через `calcBlankSellPrice(cost, totalElements, params)`
5. Это и есть `element_price_per_unit` (авто)

Менеджер может переписать это значение вручную на шаге 5 визарда.

Если reference blank не задан в настройках, поле `element_price_per_unit` обязательно для ручного ввода.

### Production hours (часы производства)

```
assemblyHours = quantity / cord.assembly_speed   // нанизывание букв + привязка шнура
packagingHours = packaging ? quantity / packaging.assembly_speed : 0
```

Карабин не добавляет отдельного времени сборки — он крепится в процессе нанизывания (входит в `cord.assembly_speed`).

Эти часы добавляются к `hoursHardwareTotal` в `calculateProductionLoad`.

### Интеграция в calculateOrderSummary

Функции `calculateOrderSummary` и `calculateProductionLoad` получают новый параметр `pendantItems`. Подвесы добавляют свои `totalCost` и `totalRevenue` к общим итогам заказа. Внутри `_buildOrderItems()` подвесы сериализуются из `this.pendants[]` в order_items с `item_type: 'pendant'`.

## Что не меняется

- Существующие product items, hardware, packaging — работают как раньше
- Лимит 6 product items не затрагивается
- Все остальные страницы (orders, order-detail, production-plan, analytics) — без изменений, кроме отображения подвесов в смете
- Подвес может сосуществовать с любыми другими позициями в заказе

## Ограничения первой версии

- Спецэлементы: только ❤️ и 😊 (можно расширить позже)
- Все элементы одного подвеса считаются по одной цене из бланков
- Нет автоматической связи с конкретным бланком буквы (кириллица/латиница) — цена берётся общая
