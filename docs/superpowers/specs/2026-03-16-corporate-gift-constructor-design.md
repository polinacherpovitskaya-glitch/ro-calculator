# Corporate Gift Constructor — Design Spec

## Overview

Standalone static page — конструктор корпоративных подарков (подвесов с буквами) для B2B-клиентов RO. Сотрудник компании-заказчика заходит по ссылке, собирает свой кастомный подвес, вводит данные для получения — заказ падает в Google Sheet.

Для каждого клиента — своя копия с отдельным `config.json` (цвета, логотип, варианты доставки).

## Approach

**Подход B: Мини-SPA с конфигом.** Статические файлы + `config.json`. Хостинг на GitHub Pages. Кастомизация под клиента — правка JSON, не кода.

## UX Flow

Одностраничный скролл. Визуализация подвеса — sticky-элемент, всегда видна при прокрутке. Шаги управления расположены под ней.

### Шаг 1 — Приветствие
- Логотип клиента из `config.clientLogo` + тёплый текст из `config.greeting`
- Подтекст из `config.subtext`

### Шаг 2+3 — Ввод имени и выбор цветов (единая секция)
- Текстовое поле: "Введи своё слово"
- Ограничения: от `config.minLetters` до `config.maxLetters` символов
- Допустимы кириллица и латиница одновременно (можно миксовать в одном слове, например "Poлина"). Валидация: только буквы A-Z, a-z, А-Я, а-я, Ё, ё — без цифр, пробелов и спецсимволов
- Визуализация появляется сразу при вводе первой буквы
- **Пустое состояние** (до ввода): в SVG-области — силуэт шнура с карабином и сердечком, подпись "Введи своё слово выше"
- Под каждой буквой в визуализации — цветные точки-свотчи из `config.letterColors`
- Клик по точке — буква меняет цвет. По дефолту все буквы первого цвета
- Счётчик: "3 из 8 букв"

### Шаг 4 — Выбор шнура
- Карточки-свотчи из `config.cordColors`
- Шнур в визуализации меняется в реальном времени

### Шаг 5 — Выбор карабина
- Карточки-свотчи из `config.carabinerColors`
- Бублик-карабин в визуализации меняет цвет

### Шаг 6 — Превью
- Итоговая визуализация: карабин → шнур → буквы → сердечко с логотипом
- Текст: "Вот твой подвес!"

### Шаг 7 — Данные получателя
- ФИО (обязательное, текст)
- Телефон (обязательное, формат свободный — принимаем любой, очищаем до цифр при сохранении)
- Башня (выпадающий список из `config.delivery.buildings`)
- Этаж (выпадающий список из `config.delivery.floors`)
- Отдел (выпадающий список из `config.delivery.departments`)

### Шаг 8 — Отправка
- Кнопка "Отправить заказ"
- Валидация: слово введено (>= minLetters), ФИО заполнено, телефон заполнен, башня/этаж/отдел выбраны
- **Успех:** "Спасибо! Твой подвес уже в работе"
- **Ошибка сети / Apps Script:** "Не удалось отправить заказ. Попробуй ещё раз" + кнопка "Повторить". Данные формы сохраняются, повторный клик отправляет снова
- Данные → Google Sheet через Apps Script

## Visualization (SVG)

Горизонтальная SVG-область, sticky при скролле, обновляется в реальном времени.

**Пустое состояние:** силуэт (серый карабин-бублик, серая линия-шнур, серое сердечко). Подпись "Введи своё слово".

**Заполненное состояние, слева направо:**
1. **Карабин** — бублик (овал с вырезом), цвет из `config.carabinerColors` по выбору пользователя
2. **Шнур** — двойная линия, catenary-кривая с лёгким провисанием между буквами, цвет из `config.cordColors`
3. **Буквы** — шрифт RO, каждая "висит" на шнуре (петелька сверху). Каждая своего цвета из `config.letterColors`. Слегка разная высота — как реальные подвески
4. **Сердечко** — всегда в конце. Форма сердца `config.charmColor`, внутри `config.charmLogo` белым

Стиль: чистый, графичный. Тонкая обводка, яркие заливки, лёгкая тень. Масштабируется по ширине экрана (responsive).

## Google Sheets Integration

### Механика
- Google Apps Script привязан к таблице, задеплоен как Web App ("Anyone" access)
- Принимает POST-запрос с JSON-данными
- Пишет строку в таблицу
- **CORS:** Apps Script Web App при `doPost` возвращает `ContentService.createTextOutput(JSON.stringify(result))`. Клиент отправляет через `fetch` в режиме `no-cors` (response нечитаем). Для определения успеха используем оптимистичный UI: показываем "Спасибо" сразу, а при сетевой ошибке (fetch reject) — показываем ошибку с кнопкой повтора.

### Автоматические подсчёты в таблице
- Итого заказов (строк)
- Итого букв (сумма столбца "Кол-во букв")
- Итого сердечек (= кол-во заказов)

## Styling

- Стилистика RO: яркая, тёплая, современная
- Основной цвет: `#FC4400` (оранжевый RO)
- Шрифт интерфейса: Inter (как в основном проекте)
- Шрифт букв подвеса: кастомный от RO
- Адаптивный дизайн (mobile-first — сотрудники будут открывать с телефона)
- Анимации: плавное появление визуализации, мягкие transitions при смене цветов

## Deployment

Для каждого нового клиента: копируем папку `corporate-gift/`, меняем `config.json` и ассеты (логотипы). Деплоим как поддиректорию на GitHub Pages или на отдельный поддомен. Автоматизация деплоя — out of scope, пока ручной процесс.

## Out of Scope (следующие этапы)
- Генерация наклеек для конвертов (PDF)
- Интеграция с основным RO-калькулятором / Supabase
- Автоматическая загрузка заказов в производство
- Админ-панель для управления конфигами клиентов
- Фотореалистичная визуализация
- Автоматизация деплоя под клиентов
